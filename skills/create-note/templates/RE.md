## RE vs ARE decision

- **RE** = user-requested report (analysis, evaluation, audit, research). Strategic deliverables.
- **ARE** = agent-generated report (review artifacts, transcript reviews, inventories, preprocessing outputs). Byproduct of a larger task.

Test: did the user specifically ask for this report? If not, use ARE prefix.

## Section structure

Reports are flexible by nature. Use the structure that fits the content. Common patterns:

### Analysis/Evaluation
1. **Summary** - 2-3 sentence conclusion (inverted pyramid)
2. **Findings** - organized by theme or category
3. **Recommendations** - numbered, actionable
4. **Methodology** - brief description of how analysis was conducted

### Audit
1. **Summary** - scope and key findings count
2. **Inventory** - table or list of items audited
3. **Issues** - categorized (critical, warning, info)
4. **Recommendations** - prioritized actions

### Research
1. **Summary** - what was researched and key takeaways
2. **Sources** - what was consulted
3. **Findings** - organized by question or theme
4. **Implications** - how this affects current work

## Writing rules

- Lead with conclusions. Reader who stops after the summary still got the point.
- Tables over prose for inventories and comparisons.
- Bold key findings and numbers.
- No hedging about observed facts.
